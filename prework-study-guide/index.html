HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • When working with visible HTML elements, content is contained between an opening tag and closing tag.
  • Children elements are elements that are nested inside other elements, or parent elements.
  • Elements and tags are used interchangeably in the developer world. However, element refers to the entire element from opening tag to closing tag. Tag refers only to what appears inside the angle (<>) brackets.
  • Semantic HTML elements indicate the role of their content and gives structure to the content of the webpage.

CSS

  • A margin indicates how mcuh space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside tan element.
  • 3 ways to style a webpage using CSS are inline CSS, internal stylesheet, and external stylesheet.
  • The most common is external stylesheet: which allows delevopers to keep all their CSS rules in a separate file.
  • When using an external stylesheet, we link the file using a element placed in the HTML .
  • * = a wild card symbol, which is used when you want to apply a rule to all the elements visible on the webpage.
  • Do Not Repeat Yourself (DRY) code: group two or more elements together by using a comma rather than repeating a rule.

Git

  • git status: checks what branch we are currently on.
  • git checkout -b branch-name: creates a new branch and switches to it.
  • git add -A: indicates we want to add all changes.
  • git pull origin main: pulls the lastest version of the project on your local computer, making sure the local branach has synced with the remote base branch.
  • git push origin main: pushes the changes we have locally to our remote GitHub branch.
